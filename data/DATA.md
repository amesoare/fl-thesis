# Data Setup

VFL-MTL experiments use two clinical databases — MIMIC-III (primary) and eICU-CRD
(external validity). Both are hosted on PhysioNet under a Data Use Agreement and are
not included in this repository.

All commands below assume you are working from the `vfl_mlt/` root directory.

---

## Expected directory layout after setup

```
data/
├── mimic-iii-clinical-database-1.4/   <- raw MIMIC-III (PhysioNet download)
├── mimic3-benchmarks/                 <- YerevaNN pipeline output
│   └── data/
│       ├── root/                      <- per-subject episode CSVs
│       ├── in-hospital-mortality/
│       │   ├── train/
│       │   ├── train_listfile.csv
│       │   ├── val_listfile.csv
│       │   └── test_listfile.csv
│       ├── decompensation/            (same layout)
│       ├── length-of-stay/            (same layout, not used in VFL training)
│       └── phenotyping/               (same layout)
├── vertical_splits/                   <- generated by data_prep scripts
│   ├── site_A_vitals.csv
│   ├── site_B_labs.csv
│   ├── site_C_composite.csv
│   └── aligned_patient_ids.csv
├── eicu-crd-2.0/                      <- raw eICU-CRD (PhysioNet download)
├── eicu_root/                         <- eICU benchmark extraction output
│   └── eicu_all_data.csv
└── eicu_vertical_splits/              <- generated by data_prep scripts
    ├── site_A_eicu.csv
    ├── site_B_eicu.csv
    ├── site_C_eicu.csv
    └── aligned_patient_ids_eicu.csv
```

## PhysioNet access (required for both datasets)

1. Register at https://physionet.org/register/ (institutional email preferred).
2. Complete the CITI "Data or Specimens Only Research" training at
   https://about.citiprogram.org/ (approximately 2–3 hours).
3. Upload your CITI completion certificate to your PhysioNet profile under **Training**.

MIMIC-III additionally requires a credentialing step:

4. Go to https://physionet.org/content/mimiciii/1.4/ and click **Request access**
   to sign the Data Use Agreement electronically.

eICU-CRD credentialing:

5. Go to https://physionet.org/content/eicu-crd/2.0/ and click **Request access**.

## MIMIC-III Clinical Database v1.4

### Step 1 — Download

```bash
mkdir -p data/mimic-iii-clinical-database-1.4

# Replace USERNAME with your PhysioNet username.
# CHARTEVENTS.csv is ~33 GB; use tmux or nohup on a remote machine.
wget -r -N -c -np \
    --user USERNAME --ask-password \
    -P data/ \
    https://physionet.org/files/mimiciii/1.4/
```

Disk: approximately 40 GB compressed.

### Step 2 — Run the YerevaNN benchmark pipeline

```bash
git clone https://github.com/YerevaNN/mimic3-benchmarks.git

cd mimic3-benchmarks
# Skip tensorflow/Keras (TF 1.x unavailable on Python ≥3.9; not needed for preprocessing)
grep -vE "tensorflow|Keras|numpy" requirements.txt | pip install -r /dev/stdin --prefer-binary
cd ..

MIMIC_RAW=data/mimic-iii-clinical-database-1.4
BENCH_OUT=data/mimic3-benchmarks/data

# Extract per-subject timeseries (30–60 min)
python -m mimic3benchmark.scripts.extract_subjects \
    $MIMIC_RAW $BENCH_OUT/root/

# Remove implausible values
python -m mimic3benchmark.scripts.validate_events \
    $BENCH_OUT/root/

# Extract per-ICU-stay episodes
python -m mimic3benchmark.scripts.extract_episodes_from_subjects \
    $BENCH_OUT/root/

# Create train / test split
python -m mimic3benchmark.scripts.split_train_and_test \
    $BENCH_OUT/root/

# Create task datasets
python -m mimic3benchmark.scripts.create_in_hospital_mortality \
    $BENCH_OUT/root/ $BENCH_OUT/in-hospital-mortality/

python -m mimic3benchmark.scripts.create_decompensation \
    $BENCH_OUT/root/ $BENCH_OUT/decompensation/

python -m mimic3benchmark.scripts.create_length_of_stay \
    $BENCH_OUT/root/ $BENCH_OUT/length-of-stay/

python -m mimic3benchmark.scripts.create_phenotyping \
    $BENCH_OUT/root/ $BENCH_OUT/phenotyping/
```

On Snellius, use `sbatch slurm/preprocess_mimic.sh` instead.

### Step 3 — Create the vertical split

```bash
python data_prep/vertical_split.py \
    --root   data/mimic3-benchmarks/data/ \
    --output data/vertical_splits/
```

This partitions 14 of the 17 YerevaNN variables across three simulated hospital
sites.

| File | Features | Task label |
|------|----------|------------|
| `site_A_vitals.csv` | HR, SBP, DBP, Temp, SpO2, RespRate, GCS Total (7 vars) | In-hospital mortality (binary, 48 h) |
| `site_B_labs.csv` | Glucose, pH, FiO2, CapRefill (4 vars) | Decompensation (binary, 24 h) |
| `site_C_composite.csv` | Height, Weight, MeanBP (3 vars) | Phenotyping (25 ICD-9 codes, multi-label) |

### Step 4 — Align patient sets (PSI simulation)

```bash
python data_prep/psi_alignment.py \
    --site_a data/vertical_splits/site_A_vitals.csv \
    --site_b data/vertical_splits/site_B_labs.csv \
    --site_c data/vertical_splits/site_C_composite.csv \
    --output data/vertical_splits/aligned_patient_ids.csv
```

`psi_alignment.py` simulates Private Set Intersection using SHA-256-hashed patient IDs,
computes the intersection across all three sites, and applies iterative stratification
(Sechidis et al., 2011) to produce balanced train / val / test splits across all 27
binary label signals (IHM + Decompensation + 25 phenotype codes).

### Step 5 — Verify

```bash
python data_prep/verify_workspace.py
```

Checks that all four task directories are non-empty, vertical splits are present,
`aligned_patient_ids.csv` covers at least 30,000 patients, feature columns do not
overlap between site files, and label balance is within acceptable range for each task.

## eICU Collaborative Research Database v2.0

### Step 1 — Download

```bash
mkdir -p data/eicu-crd-2.0

# Store your PhysioNet password in ~/.physionet_password (chmod 600).
wget -r -N -c -np \
    --user USERNAME \
    --password "$(cat ~/.physionet_password)" \
    -P data/ \
    https://physionet.org/files/eicu-crd/2.0/
```

On Snellius, use `sbatch slurm/run_eicu_download.sh` instead.

Disk: approximately 7 GB compressed.

### Step 2 — Extract per-patient timeseries

```bash
mkdir -p data/eicu_root

python data_prep/eicu_benchmark/data_extraction/data_extraction_root.py \
    --eicu_dir  data/eicu-crd-2.0/ \
    --output_dir data/eicu_root/
```

On Snellius, use `sbatch slurm/run_eicu_extraction.sh`. The extraction writes large
intermediates to `$TMPDIR` (SLURM scratch) and copies only `eicu_all_data.csv` to home.

### Step 3 — Create the vertical split

```bash
python data_prep/eicu_vertical_split.py \
    --root_dir data/eicu_root/ \
    --eicu_dir data/eicu-crd-2.0/ \
    --output   data/eicu_vertical_splits/ \
    --seed 42
```

| File | Features | Task label |
|------|----------|------------|
| `site_A_eicu.csv` | HR, SBP, DBP, MAP, SpO2, RespRate, Temp (7 vars) | In-hospital mortality (binary) |
| `site_B_eicu.csv` | Glucose, pH, FiO2 (3 vars) | Remaining length of stay (regression, days) |
| `site_C_eicu.csv` | GCS Total, Height, Weight (3 vars) | Phenotyping (25 ICD-9 codes, multi-label) |

### Step 4 — Align patient sets (PSI simulation)

```bash
python data_prep/eicu_psi_alignment.py \
    --site_a data/eicu_vertical_splits/site_A_eicu.csv \
    --site_b data/eicu_vertical_splits/site_B_eicu.csv \
    --site_c data/eicu_vertical_splits/site_C_eicu.csv \
    --output data/eicu_vertical_splits/aligned_patient_ids_eicu.csv
```

On Snellius, Steps 3 and 4 are combined in `sbatch slurm/run_eicu_vertical_split.sh`.

## Synthetic mode (no data required)

All experiment scripts accept `--use_synthetic` to run a smoke test with randomly
generated data. This mode is useful for verifying installation but does not reproduce
any reported results.

```bash
python train.py --rounds 5 --use_synthetic
```
